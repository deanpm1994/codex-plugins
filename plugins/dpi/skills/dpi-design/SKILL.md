# DPI Design

Work through the design phase one decision at a time until the shape of the session is clear.

- Ask one question at a time and wait for the response before moving on.
- If the answer can be found in the repo, inspect it first.
- Write exactly one spec for the session under `docs/dpi/specs/`.
- End the spec with a handoff section that captures the current decision, open questions, next phase, and editable notes.
- Do not start planning until the user approves the spec.
