# DPI

Use this skill to guide the user through the full Design -> Plan -> Implement flow.

- Start at design when the user wants the full flow.
- Allow direct entry into design, plan, implement, or handoff when the user wants a specific phase.
- Keep one spec per session, but allow multiple plan docs when a session needs split implementation slices.
- Record phase boundaries inside the active spec or plan doc only.
- Use the `dpi-handoff` skill whenever you need to summarize the transition or hand the work off to the next phase.
