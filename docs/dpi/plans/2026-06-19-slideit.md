# Slideit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `slideit` Codex plugin that turns outlines, notes, or ChatGPT/search transcripts into polished Reveal.js HTML decks with Excalidraw-backed diagrams and publishes the plugin to the personal marketplace.

**Architecture:** Ship one standalone plugin under `plugins/slideit` with a small skill tree, a deck-generation pipeline, and a thin asset/diagram layer. Keep the engine boundary isolated so Reveal.js is the v1 default without hard-coding the rest of the workflow to it. Treat transcript ingestion and disclosure decisions as first-class behavior in the generator, not as a separate helper script.

**Tech Stack:** Codex plugin manifest, Markdown `SKILL.md` files, Node.js helper scripts for generation logic and image handling, Reveal.js as the slide engine, Excalidraw or Excalidraw-style assets exported through SVG-to-PNG, and the existing marketplace JSON under `.agents/plugins/marketplace.json`.

- [ ] **Step 2: Add disclosure rules**

```js
export function resolveDisclosure({ audience, source, ambiguity }) {
  if (ambiguity) {
    return { action: "ask" };
  }
  if (audience === "internal") {
    return { action: "show" };
  }
  return { action: "hide" };
}
```

Expected: the plugin can show, hide, or ask depending on context and ambiguity.

- [ ] **Step 2: Map diagrams into slide assets**

```js
export function attachDiagramAssets(slide, assets) {
  return {
    ...slide,
    assets: [...(slide.assets || []), ...assets]
  };
}
```

Expected: diagrams stay associated with slides rather than becoming orphaned files.
