# Caveman Plugin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a standalone `caveman` Codex plugin that compresses agent communication with `low`, `medium`, and `high` modes, persists defaults, and reports estimated or measured savings.

**Architecture:** Ship one standalone plugin under `plugins/caveman` with a small, explicit skill tree. The main `caveman` skill owns activation, deactivation, and level switching. Companion skills handle stats and status output so the core entrypoint stays small. Persist user preferences and savings through a dedicated state helper that can use Codex-native persistence if available, or a local JSON fallback if not.

**Tech Stack:** Codex plugin scaffold, Markdown `SKILL.md` files, JSON manifest, lightweight Node scripts for state and savings math, `python3` validation helpers from the plugin-creator skill.

- [ ] **Step 1: Implement the state adapter**

```js
export async function loadState() {}
export async function saveState(nextState) {}
export function defaultState() {
  return {
    enabled: false,
    mode: "medium",
    lifetime: {
      estimatedTokensSaved: 0,
      estimatedMoneySavedUsd: 0
    }
  };
}
```

Expected: the helper can read and write caveman preferences and totals, using a persistent Codex surface when available and falling back to a local JSON file otherwise.

- [ ] **Step 2: Implement savings calculations**

```js
export function estimateSavings(beforeTokens, afterTokens, usdPerToken) {
  return {
    tokensSaved: Math.max(0, beforeTokens - afterTokens),
    moneySavedUsd: Math.max(0, (beforeTokens - afterTokens) * usdPerToken)
  };
}
```

Expected: the calculator produces deterministic numbers for both estimated and measured paths.

- [ ] **Step 3: Add statusline formatting**

```js
export function renderStatus(state) {
  return `caveman:${state.enabled ? state.mode : "off"} saved:${state.lifetime.estimatedTokensSaved}t $${state.lifetime.estimatedMoneySavedUsd.toFixed(2)}`;
}
```

Expected: the statusline output is short enough to scan quickly and includes mode plus savings.

- [ ] **Step 4: Add unit tests**

```js
import test from "node:test";
import assert from "node:assert/strict";
import { defaultState } from "../scripts/state.mjs";
import { estimateSavings } from "../scripts/savings.mjs";

test("defaultState returns caveman defaults", () => {
  assert.equal(defaultState().mode, "medium");
});

test("estimateSavings clamps negative savings to zero", () => {
  assert.deepEqual(estimateSavings(100, 120, 0.01), {
    tokensSaved: 0,
    moneySavedUsd: 0
  });
});
```

Expected: `node --test` passes once the helper functions are implemented.
